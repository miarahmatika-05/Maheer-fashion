**Keputusan Arsitektur Kunci:**

- **Next.js App Router** dipilih untuk mendukung SSR/SSG yang memungkinkan halaman dashboard dan inventaris di-render di server sebelum dikirim ke browser, meminimalkan layout shift.
- **Supabase sebagai BaaS** menghilangkan kebutuhan server backend terpisah вҖ” Auth, Database, Realtime, dan Storage tersedia dalam satu layanan terkelola.
- **Row Level Security (RLS)** diimplementasikan langsung di layer database sehingga pembatasan akses role tidak bergantung pada frontend вҖ” menghindari bypass melalui manipulasi API request.

## 2. Layer-by-Layer Breakdown

### 2.1 Client Layer

Browser modern (Chrome v110+, Firefox v109+, Edge v110+, Safari v16+) pada perangkat desktop atau tablet dengan resolusi minimum 1280Г—720px.

### 2.2 Frontend / UI Layer

| Komponen | Teknologi | Peran |
|---|---|---|
| Routing & Rendering | Next.js 14 App Router | File-based routing, SSR, API routes |
| UI State | React 18 (hooks) | Komponen interaktif, state lokal |
| Styling | Tailwind CSS 3.x | Utility-first, tema konsisten |
| Komponen Aksesibel | Radix UI + Base UI | Dialog, Dropdown, Combobox headless |
| Animasi | Framer Motion | Page transitions, micro-interactions |
| Grafik | Recharts | Line chart, Pie chart, Bar chart pada dashboard |
| HTTP Client | Supabase JS SDK | Semua komunikasi ke backend |

### 2.3 Data & Auth Layer

| Fitur Supabase | Penggunaan dalam Aplikasi |
|---|---|
| **PostgreSQL** | Penyimpanan semua data (users, products, transactions) |
| **Supabase Auth** | Login/logout, JWT session, refresh token |
| **PostgREST** | REST API otomatis untuk semua tabel database |
| **Row Level Security** | Pembatasan read/write berdasarkan `auth.uid()` dan kolom `role` |
| **Realtime** | Subscribe perubahan stok dan transaksi baru secara live |

### 2.4 Deployment Layer

- **Hosting:** Vercel (GitHub integration untuk CI/CD otomatis).
- **Branching strategy:** `main` вҶ’ production, `develop` вҶ’ staging, feature branches вҶ’ preview URLs.
- **Environment:** Production, Staging, dan Development menggunakan proyek Supabase yang terpisah.

### 3.2 Definisi Tabel Lengkap

#### Tabel `users`

```sql
CREATE TABLE users (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email       VARCHAR(255) UNIQUE NOT NULL,
  full_name   VARCHAR(100) NOT NULL,
  role        role_enum NOT NULL DEFAULT 'cashier',
  is_active   BOOLEAN NOT NULL DEFAULT true,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TYPE role_enum AS ENUM ('admin', 'cashier');
```

#### Tabel `products`

```sql
CREATE TABLE products (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  sku              VARCHAR(50) UNIQUE NOT NULL,
  name             VARCHAR(200) NOT NULL,
  category         VARCHAR(100),
  size             VARCHAR(10),   -- S, M, L, XL, XXL, dll.
  color            VARCHAR(50),
  current_stock    INTEGER NOT NULL DEFAULT 0 CHECK (current_stock >= 0),
  initial_stock    INTEGER NOT NULL DEFAULT 0,
  sell_price       DECIMAL(12,2) NOT NULL CHECK (sell_price >= 0),
  cost_price       DECIMAL(12,2) NOT NULL CHECK (cost_price >= 0),
  low_stock_threshold INTEGER NOT NULL DEFAULT 5,
  is_active        BOOLEAN NOT NULL DEFAULT true,
  created_at       TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at       TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- Index untuk performa pencarian SKU dan nama produk
CREATE INDEX idx_products_sku ON products(sku);
CREATE INDEX idx_products_name ON products(name);
CREATE INDEX idx_products_category ON products(category);
CREATE INDEX idx_products_is_active ON products(is_active);
```

#### Tabel `transactions`

```sql
CREATE TABLE transactions (
  id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  created_at     TIMESTAMPTZ NOT NULL DEFAULT now(),
  channel        channel_enum NOT NULL,
  total_revenue  DECIMAL(12,2) NOT NULL DEFAULT 0,
  total_cost     DECIMAL(12,2) NOT NULL DEFAULT 0,
  total_profit   DECIMAL(12,2) GENERATED ALWAYS AS (total_revenue - total_cost) STORED,
  notes          TEXT,
  cashier_id     UUID NOT NULL REFERENCES users(id),
  customer_name  VARCHAR(100)
);

CREATE TYPE channel_enum AS ENUM ('offline', 'shopee', 'tiktok_shop');

-- Index untuk laporan dan filter tanggal
CREATE INDEX idx_transactions_created_at ON transactions(created_at);
CREATE INDEX idx_transactions_channel ON transactions(channel);
CREATE INDEX idx_transactions_cashier_id ON transactions(cashier_id);
```

#### Tabel `transaction_details`

```sql
CREATE TABLE transaction_details (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  transaction_id  UUID NOT NULL REFERENCES transactions(id) ON DELETE CASCADE,
  product_id      UUID NOT NULL REFERENCES products(id),
  quantity        INTEGER NOT NULL CHECK (quantity > 0),
  unit_price      DECIMAL(12,2) NOT NULL,  -- Snapshot harga saat transaksi
  unit_cost       DECIMAL(12,2) NOT NULL,  -- Snapshot HPP saat transaksi
  subtotal        DECIMAL(12,2) GENERATED ALWAYS AS (quantity * unit_price) STORED,
  profit          DECIMAL(12,2) GENERATED ALWAYS AS (quantity * (unit_price - unit_cost)) STORED
);

CREATE INDEX idx_txn_details_transaction_id ON transaction_details(transaction_id);
CREATE INDEX idx_txn_details_product_id ON transaction_details(product_id);
```

### 3.3 Row Level Security (RLS) Policies

```sql
-- Aktifkan RLS pada semua tabel
ALTER TABLE users ENABLE ROW LEVEL SECURITY;
ALTER TABLE products ENABLE ROW LEVEL SECURITY;
ALTER TABLE transactions ENABLE ROW LEVEL SECURITY;
ALTER TABLE transaction_details ENABLE ROW LEVEL SECURITY;

-- Policy: Admin dapat melihat semua data
CREATE POLICY "admin_select_all_products" ON products
  FOR SELECT USING (
    (SELECT role FROM users WHERE id = auth.uid()) = 'admin'
  );

-- Policy: Kasir hanya bisa SELECT produk aktif
CREATE POLICY "cashier_select_active_products" ON products
  FOR SELECT USING (
    is_active = true AND
    (SELECT role FROM users WHERE id = auth.uid()) = 'cashier'
  );

-- Policy: Hanya admin yang bisa INSERT/UPDATE/DELETE produk
CREATE POLICY "admin_manage_products" ON products
  FOR ALL USING (
    (SELECT role FROM users WHERE id = auth.uid()) = 'admin'
  );

-- Policy: Kasir hanya bisa melihat transaksinya sendiri
CREATE POLICY "cashier_own_transactions" ON transactions
  FOR SELECT USING (cashier_id = auth.uid());

-- Policy: Admin bisa melihat semua transaksi
CREATE POLICY "admin_all_transactions" ON transactions
  FOR SELECT USING (
    (SELECT role FROM users WHERE id = auth.uid()) = 'admin'
  );
```

## 5. API Specification (Ringkasan)

Semua endpoint dihasilkan secara otomatis oleh **PostgREST** (bawaan Supabase). Base URL: `https://<PROJECT_REF>.supabase.co/rest/v1`

Header wajib pada setiap request:

```
Authorization: Bearer <JWT_TOKEN>
apikey: <SUPABASE_ANON_KEY>
Content-Type: application/json
```

## 6. State Management

Aplikasi tidak menggunakan state management library eksternal (Redux/Zustand). State dikelola dengan strategi:

| Jenis State | Pendekatan | Contoh |
|---|---|---|
| **Server State** | React Query / Next.js `fetch` + cache | Daftar produk, histori transaksi |
| **UI State lokal** | `useState` / `useReducer` | Keranjang POS, modal terbuka |
| **Auth State** | Supabase Auth context | Sesi pengguna aktif, role |
| **Realtime State** | Supabase `channel.on()` | Update stok live |

## 7. Real-time Subscriptions

```typescript
// Contoh: Subscribe perubahan stok produk
const channel = supabase
  .channel('stock-changes')
  .on(
    'postgres_changes',
    { event: 'UPDATE', schema: 'public', table: 'products' },
    (payload) => {
      // Update UI secara otomatis ketika stok berubah
      updateProductStock(payload.new.id, payload.new.current_stock);
    }
  )
  .subscribe();

// Cleanup on unmount
return () => { supabase.removeChannel(channel); };
```

Realtime diaktifkan untuk:
- Tabel `products` (event: `UPDATE`) вҖ” notifikasi perubahan stok.
- Tabel `transactions` (event: `INSERT`) вҖ” dashboard admin menampilkan transaksi baru secara langsung.

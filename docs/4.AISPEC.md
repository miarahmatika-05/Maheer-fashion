## 1. AI Prompt Registry

### 1.1 Konvensi Penamaan Prompt

Format: `[MODUL]_[FUNGSI]_v[VERSION]`

Contoh: `INV_RESTOCK_SUGGEST_v1`, `POS_SEARCH_ENHANCE_v1`

Setiap prompt yang diubah **wajib menaikkan versi** dan mendokumentasikan perubahan di tabel ini.

### 1.2 Registry Prompt Aktif

---

#### PROMPT: `INV_RESTOCK_SUGGEST_v1`

**Tujuan:** Menyarankan jumlah restock optimal berdasarkan histori penjualan.

**Model:** `gpt-4o-mini` / `claude-3-haiku` (tergantung provider aktif)

**System Prompt:**

```
Anda adalah asisten manajemen inventaris untuk toko fashion "Maheer Fashion".
Tugas Anda adalah menganalisis data penjualan historis dan memberikan saran restock yang tepat.

Aturan:
- Berikan saran dalam Bahasa Indonesia yang singkat dan jelas.
- Format output selalu dalam JSON yang valid.
- Pertimbangkan tren musiman jika ada.
- Jangan merekomendasikan stok lebih dari 3x rata-rata penjualan mingguan.
```

**User Prompt Template:**

```
Berikut data penjualan 30 hari terakhir untuk SKU {sku}:
- Total terjual: {units_sold} unit
- Rata-rata per hari: {avg_daily} unit
- Stok saat ini: {current_stock} unit
- Lead time supplier: {lead_time_days} hari

Berikan saran jumlah restock yang optimal dalam format JSON:
{"recommended_qty": <number>, "reasoning": "<string>", "urgency": "low|medium|high"}
```

**Output Schema:**

```json
{
  "recommended_qty": 50,
  "reasoning": "Rata-rata penjualan 5 unit/hari dengan lead time 7 hari, buffer safety stock 2 minggu.",
  "urgency": "high"
}
```

---

#### PROMPT: `DASH_INSIGHT_GENERATE_v1`

**Tujuan:** Menghasilkan insight naratif dari data dashboard untuk Admin.

**Model:** `claude-3-haiku` / `gpt-4o-mini`

**System Prompt:**

```
Anda adalah analis bisnis senior untuk toko fashion retail Indonesia.
Tugas Anda adalah membaca data penjualan dan memberikan insight bisnis yang actionable.

Aturan:
- Gunakan Bahasa Indonesia yang profesional namun mudah dipahami.
- Maksimal 3 poin insight utama.
- Setiap insight harus disertai saran tindakan konkret.
```

**User Prompt Template:**

```
Data ringkasan toko bulan {month} {year}:
```

**Tujuan:** Meningkatkan pencarian produk menggunakan pemahaman bahasa natural.

## 2. OCR Integration

### 2.1 Use Case OCR

Fitur OCR digunakan untuk:
1. **Scan nota supplier** → otomatis mengisi form restock produk.
2. **Scan struk pembayaran pelanggan** → verifikasi pembayaran channel online.
3. **Scan label produk fisik** → identifikasi SKU dari kamera perangkat.

### 2.2 Provider OCR

**Primary:** Google Cloud Vision API  
**Fallback:** Tesseract.js (client-side, untuk kondisi offline terbatas)

### 2.3 Alur OCR untuk Nota Supplier

```
User (Admin)              Frontend              OCR Service           AI Parsing
     │                       │                      │                    │
     │── Upload foto nota ──►│                      │                    │
     │                       │── POST /api/ocr ─────►│                    │
     │                       │  (base64 image)       │── Extract text ───►│
     │                       │                      │◄── Raw text ───────│
     │                       │◄── OCR Result ────────│                    │
     │                       │                       │                    │
     │                       │── Parse dengan AI ──────────────────────►│
     │                       │  (PROMPT: OCR_PARSE_SUPPLIER_v1)          │
     │                       │◄── Structured JSON ──────────────────────│
     │                       │                                           │
     │◄── Pre-filled form ───│                                           │
     │── Konfirmasi/edit ───►│                                           │
     │                       │── Simpan ke database                      │
```

### 2.4 PROMPT: `OCR_PARSE_SUPPLIER_v1`

**System Prompt:**

```
Anda adalah sistem ekstraksi data dari nota pembelian supplier fashion Indonesia.
Tugas Anda adalah mengekstrak informasi produk dari teks hasil OCR nota supplier.

Aturan:
- Output SELALU dalam format JSON valid, tanpa teks tambahan.
- Jika field tidak ditemukan, gunakan null.
- Normalisasi ukuran ke format: S, M, L, XL, XXL.
- Normalisasi warna ke Bahasa Indonesia standar.
- Jumlah unit selalu berupa integer positif.
```

**User Prompt Template:**

```
Teks dari nota supplier berikut:
---
{ocr_raw_text}
---

Ekstrak semua item produk dalam format JSON array:
[
  {
    "product_name": string,
    "sku_hint": string | null,
    "size": string | null,
    "color": string | null,
    "quantity": integer,
    "unit_cost": number | null
  }
]
```

### 2.5 Spesifikasi Teknis OCR

| Parameter | Nilai |
|---|---|
| Format gambar diterima | JPEG, PNG, WEBP |
| Ukuran maksimum file | 10 MB |
| Resolusi minimum | 300 DPI (rekomendasi) |
| Bahasa OCR | Indonesia + Inggris |
| Timeout API | 30 detik |
| Retry logic | 3x dengan exponential backoff |

### 2.6 API Route OCR

```typescript
// app/api/ocr/route.ts
export async function POST(request: Request) {
  const { imageBase64, mimeType } = await request.json();

  // Validasi input
  if (!imageBase64 || !mimeType) {
    return Response.json({ error: 'Missing required fields' }, { status: 400 });
  }

  // Call Google Cloud Vision API
  const visionResponse = await fetch(
    `https://vision.googleapis.com/v1/images:annotate?key=${process.env.GOOGLE_VISION_API_KEY}`,
    {
      method: 'POST',
      body: JSON.stringify({
        requests: [{
          image: { content: imageBase64 },
          features: [{ type: 'DOCUMENT_TEXT_DETECTION' }]
        }]
      })
    }
  );

  const { responses } = await visionResponse.json();
  const rawText = responses[0]?.fullTextAnnotation?.text || '';

  // Parse dengan AI
  const parsedItems = await parseOCRWithAI(rawText);

  return Response.json({ rawText, parsedItems });
}
```

## 3. WhatsApp Bot Flow

### 3.1 Gambaran Umum

WhatsApp Bot terintegrasi dengan sistem untuk memberikan notifikasi otomatis dan kemampuan query sederhana via WhatsApp kepada Admin.

**Provider:** Twilio WhatsApp API / WhatsApp Business Cloud API (Meta)

### 3.2 Trigger Notifikasi Otomatis (Outbound)

| Trigger | Penerima | Pesan |
|---|---|---|
| Stok produk mencapai threshold | Admin | "⚠️ Stok rendah: [SKU] - [Nama Produk] tersisa [X] unit" |
| Transaksi > Rp 1.000.000 | Admin | "💰 Transaksi besar: Rp [amount] via [channel] oleh [kasir]" |
| Laporan harian (19:00 WIB) | Admin | Ringkasan: total transaksi, revenue, profit hari ini |
| Login dari IP baru | Admin | "🔐 Login baru terdeteksi dari IP [x.x.x.x] oleh [user]" |
| Approval request baru | Approver | "🔔 Ada permintaan persetujuan dari [requester]: [action]" |

```

     │                           │                          │

     │                           │── Query low stock ───────►│
```

### 3.4 Daftar Command Bot

| Command | Deskripsi | Contoh Response |
|---|---|---|
| `stok` / `stok rendah` | Daftar produk stok di bawah threshold | List SKU + stok saat ini |
| `laporan hari ini` | Ringkasan transaksi hari ini | Total txn, revenue, profit |
| `laporan kemarin` | Ringkasan transaksi kemarin | Total txn, revenue, profit |
| `transaksi terakhir` | 5 transaksi terbaru | ID, amount, channel, kasir |
| `help` / `bantuan` | Daftar command yang tersedia | Menu lengkap |

### 3.6 Webhook Handler

```typescript
// app/api/whatsapp/webhook/route.ts
export async function POST(request: Request) {
  const body = await request.json();
  const message = body.entry?.[0]?.changes?.[0]?.value?.messages?.[0];

  if (!message) return Response.json({ status: 'ok' });

  // Validasi pengirim adalah Admin terdaftar
  const admin = await getAdminByPhone(message.from);
  if (!admin) {
    await sendWhatsAppMessage(message.from, 'Akses tidak diizinkan.');
    return Response.json({ status: 'unauthorized' });
  }

  // Parse intent
  const intent = await parseIntent(message.text?.body || '');

  // Handle intent
  const response = await handleBotIntent(intent, admin.id);

  await sendWhatsAppMessage(message.from, response);
  return Response.json({ status: 'ok' });
}
```

## 4. Integration Contracts

### 4.1 Google Cloud Vision API

| Parameter | Detail |
|---|---|
| Base URL | `https://vision.googleapis.com/v1` |
| Auth | API Key (server-side only) |
| Rate Limit | 1800 requests/menit |
| Biaya | $1.50 per 1000 unit setelah free tier (1000/bulan) |
| Env Variable | `GOOGLE_VISION_API_KEY` |
| Error Handling | Fallback ke Tesseract.js jika timeout/error |

**Error Response Standard:**

```json
{
  "error": {
    "code": 429,
    "message": "Quota exceeded",
    "status": "RESOURCE_EXHAUSTED"
  }
}
```

### 4.2 WhatsApp Business Cloud API (Meta)

| Parameter | Detail |
|---|---|
| Base URL | `https://graph.facebook.com/v18.0` |
| Auth | Bearer Token (permanent access token) |
| Webhook Verify Token | Disimpan di env: `WA_WEBHOOK_VERIFY_TOKEN` |
| Phone Number ID | `WA_PHONE_NUMBER_ID` |
| Rate Limit | 80 messages/detik per phone number |
| Env Variables | `WA_ACCESS_TOKEN`, `WA_PHONE_NUMBER_ID`, `WA_WEBHOOK_VERIFY_TOKEN` |

**Format Kirim Pesan:**

| Parameter | Detail |

### 4.4 Kontrak Error Handling

Semua integrasi eksternal mengikuti kontrak error handling yang sama:

```typescript
interface IntegrationResult<T> {
  success: boolean;
  data?: T;
  error?: {
    code: string;
    message: string;
    provider: string;
    retryable: boolean;
  };
}

// Contoh penggunaan
const result = await callOCR(imageBase64);
if (!result.success) {
  if (result.error?.retryable) {
    // Retry dengan exponential backoff
    await retryWithBackoff(() => callOCR(imageBase64), maxRetries: 3);
  } else {
    // Log error dan tampilkan pesan ke user
    await logIntegrationError(result.error);
    throw new UserFacingError('Gagal memproses gambar. Silakan coba lagi.');
  }
}
```


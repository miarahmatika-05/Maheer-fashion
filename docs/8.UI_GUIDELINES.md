## 1. Filosofi Desain

Maheer Fashion POS dirancang dengan prinsip **"Efisiensi Kasir Pertama"** — antarmuka harus memungkinkan kasir memproses transaksi dalam < 2 menit tanpa kebingungan navigasi.

**Prinsip Desain:**

1. **Clarity over decoration** — Setiap elemen ada karena fungsinya, bukan estetika semata.
2. **Speed first** — Aksi paling sering digunakan (POS, pencarian produk) harus dapat diakses dalam maksimal 2 klik.
3. **Error prevention** — Desain mencegah kesalahan input sebelum terjadi, bukan hanya menampilkan error setelah fakta.
4. **Consistent feedback** — Setiap aksi pengguna selalu mendapat respons visual (loading, sukses, error) dalam < 300ms.
5. **Mobile-aware** — Meskipun primary target adalah tablet/desktop, elemen touch target minimal 44×44px.

## 3. Tipografi

**Font Family:** `Inter` (Google Fonts) — sans-serif yang bersih dan legibel untuk data-dense UI.

```css
/* Heading */
--text-3xl: 1.875rem / 2.25rem, font-weight: 700  → Judul halaman
--text-2xl: 1.5rem / 2rem, font-weight: 700       → Judul section, total transaksi
--text-xl:  1.25rem / 1.75rem, font-weight: 600   → Sub-judul, nama produk
--text-lg:  1.125rem / 1.75rem, font-weight: 600  → Heading kecil

/* Body */
--text-base: 1rem / 1.5rem, font-weight: 400      → Body text utama
--text-sm:   0.875rem / 1.25rem, font-weight: 400 → Caption, label form
--text-xs:   0.75rem / 1rem, font-weight: 400     → Badge, tooltip, metadata
```

**Aturan Tipografi:**

- Harga selalu ditampilkan dengan **font-weight: 700** dan ukuran lebih besar dari konteks sekitarnya.
- Kode SKU menggunakan `font-family: monospace` untuk keterbacaan.
- Teks error menggunakan `text-danger-500`, bukan hanya ikon.

## 4. Komponen UI

### 4.1 Button

```
Variant    | Penggunaan                        | Style
-----------|-----------------------------------|----------------------------------
primary    | Aksi utama (Proses, Simpan)       | bg-primary-500, text-white
secondary  | Aksi sekunder (Batal, Kembali)    | border-gray-300, text-gray-700
danger     | Aksi destruktif (Hapus)           | bg-danger-500, text-white
ghost      | Aksi ringan (Edit inline)         | text-primary-600, no background
disabled   | Aksi tidak tersedia               | opacity-50, cursor-not-allowed
```

**Size:**

```
sm  → h-8, px-3, text-sm   → Aksi dalam tabel
md  → h-10, px-4, text-sm  → Standar (default)
lg  → h-12, px-6, text-base → CTA utama (Proses Transaksi)
```

**Loading state:** Tombol menampilkan spinner dan teks berubah (contoh: "Memproses..." saat loading) — bukan disabled tanpa indikasi.

### 4.2 Input & Form

```
Normal:  border-gray-300, bg-white, focus:ring-2 ring-primary-500
Error:   border-danger-500, bg-danger-50, focus:ring-danger-500
Success: border-success-500, bg-success-50
Disabled: bg-gray-100, text-gray-400, cursor-not-allowed
```

**Aturan form:**
- Label selalu di atas input (bukan placeholder-only).
- Pesan error muncul di bawah input, bukan di atas.
- Field wajib diberi tanda `*` merah di sebelah label.
- Format Rupiah: input angka otomatis diformat dengan titik ribuan saat blur.

### 4.3 Badge / Status Indicator

```
Stok Aman:    bg-success-100 text-success-700  → "12 unit"
Stok Rendah:  bg-warning-100 text-warning-700  → "3 unit ⚠️"
Stok Habis:   bg-danger-100  text-danger-700   → "Habis"
Channel Offline:      bg-gray-100   text-gray-700   → "Offline"
Channel Shopee:       bg-orange-100 text-orange-700 → "Shopee"
Channel TikTok Shop:  bg-pink-100   text-pink-700   → "TikTok Shop"
Role Admin:           bg-primary-100 text-primary-700 → "Admin"
Role Kasir:           bg-gray-100   text-gray-700   → "Kasir"
```

### 4.4 Card

Kartu produk (POS) dan kartu inventaris menggunakan:
```
bg-white, rounded-lg (8px), shadow-sm
hover: shadow-md, border-primary-200 (highlight saat dipilih)
active: border-primary-500, bg-primary-50
```

### 4.5 Tabel

```

```

## 6. Pola Interaksi

### 6.1 Loading States

Selalu tampilkan status loading untuk operasi async:

```
API call < 300ms:  Tidak perlu spinner (optimistic UI)
API call 300ms–1s: Skeleton loader (placeholder shape)
API call > 1s:     Spinner + teks deskriptif
```

**Skeleton loader** digunakan untuk daftar produk, tabel transaksi, dan card dashboard.

### 6.2 Feedback Transaksi Berhasil

Setelah transaksi berhasil:
1. Modal sukses muncul dengan animasi scale-up (Framer Motion).
2. Tampilkan: nomor transaksi, total, channel, nama kasir.
3. Dua opsi: "Cetak Struk" dan "Transaksi Baru".
4. Auto-dismiss setelah 10 detik jika tidak ada aksi.

### 6.3 Konfirmasi Aksi Destruktif

Semua aksi destruktif (hapus produk, batal transaksi) menggunakan **Dialog konfirmasi**:

```
Judul:  "Nonaktifkan [Nama Produk]?"
Body:   "Produk ini tidak akan muncul di katalog POS.
         Histori transaksi tetap tersimpan."
Actions: [Batal]  [Ya, Nonaktifkan] (tombol danger)
```

### 6.4 Toast Notifications

Notifikasi singkat untuk feedback non-kritis:

```
Posisi:   Bottom-right
Duration: 3 detik
Types:    success (hijau), error (merah), warning (kuning), info (biru)
```

Contoh:
- `✅ Stok GM-ALZ-XL berhasil diperbarui (+50 unit)`
- `❌ Gagal menyimpan produk. SKU sudah digunakan.`
- `⚠️ Perubahan harga > 20% memerlukan persetujuan.`

### 6.5 Empty States

Setiap tampilan daftar kosong harus memiliki empty state yang informatif:

```
Ikon (ilustrasi relevan)
Judul: "Belum ada [entitas]"
Deskripsi singkat
CTA: Tombol aksi utama (jika relevan)

Contoh:
  Ikon: 📦
  Judul: "Belum ada produk"
  Deskripsi: "Tambah produk pertama untuk mulai berjualan."
  CTA: [+ Tambah Produk]
```

## 7. Aksesibilitas

### 7.1 Target Minimum

Aplikasi ini menargetkan **WCAG 2.1 Level AA**.

### 7.2 Checklist Aksesibilitas

- [ ] Semua gambar/ikon yang fungsional memiliki `alt` text atau `aria-label`.
- [ ] Kontras warna teks utama ≥ 4.5:1 (AA normal text).
- [ ] Kontras warna heading ≥ 3:1 (AA large text).
- [ ] Semua form input memiliki `<label>` yang terhubung via `htmlFor`.
- [ ] Pesan error terhubung ke input via `aria-describedby`.
- [ ] Modal menggunakan `role="dialog"` dan `aria-modal="true"`.
- [ ] Focus trap aktif saat modal terbuka.
- [ ] Navigasi keyboard berfungsi penuh (Tab, Enter, Escape, Arrow keys).
- [ ] Tombol loading menggunakan `aria-busy="true"` dan teks berubah.
- [ ] Tabel data memiliki `<th scope="col">` dan summary jika kompleks.

### 7.3 Data Attributes untuk Testing

Semua elemen interaktif yang digunakan dalam E2E testing menggunakan `data-testid`:

```tsx
<button data-testid="process-transaction-btn">Proses Transaksi</button>
<input data-testid="product-search-input" />
<div data-testid="cart-item-list" />
<select data-testid="channel-selector" />
```

## 8. Panduan per Modul

### 8.1 POS — Prioritas Visual

1. **Search bar** harus auto-focus saat halaman POS dibuka.
2. **Stok 0** — kartu produk ditampilkan dengan opacity-50 dan badge "Habis". Tidak dapat diklik.
3. **Total transaksi** di keranjang selalu visible dan update real-time.
4. **Tombol "Proses Transaksi"** hanya aktif jika keranjang tidak kosong DAN semua produk punya stok.

### 8.2 Dashboard — Prioritas Visual

1. KPI cards di atas fold — langsung terlihat tanpa scroll.
2. Grafik line chart sebagai centerpiece — ukuran minimal 400px tinggi.
3. Gunakan warna konsisten per channel di semua chart (Offline=gray, Shopee=orange, TikTok=pink).
4. Tambahkan tooltip interaktif pada semua titik data grafik.

### 8.3 Inventaris — Prioritas Visual

1. Badge stok rendah/habis harus terlihat jelas dalam tabel.
2. Filter (kategori, ukuran, status stok) selalu visible di atas tabel — bukan hidden dalam hamburger menu.
3. Tombol "Restock" diposisikan di kolom paling kanan tabel — mudah diakses tanpa membuka detail.
4. Konfirmasi wajib sebelum nonaktifkan produk.

### 8.4 Contoh Implementasi Komponen

```tsx
// Contoh: Badge status stok
interface StockBadgeProps {
  stock: number;
  threshold: number;
}

const StockBadge: FC<StockBadgeProps> = ({ stock, threshold }) => {
  if (stock === 0) return (
    <span className="inline-flex items-center px-2.5 py-0.5 rounded-full text-xs font-medium bg-red-100 text-red-700">
      Habis
    </span>
  );

  if (stock <= threshold) return (
    <span className="inline-flex items-center px-2.5 py-0.5 rounded-full text-xs font-medium bg-yellow-100 text-yellow-700">
      ⚠️ {stock} unit
    </span>
  );

  return (
    <span className="inline-flex items-center px-2.5 py-0.5 rounded-full text-xs font-medium bg-green-100 text-green-700">
      {stock} unit
    </span>
  );
};
```

```tsx
// Contoh: Format harga Rupiah
export function formatRupiah(amount: number): string {
  return new Intl.NumberFormat('id-ID', {
    style: 'currency',
    currency: 'IDR',
    minimumFractionDigits: 0,
    maximumFractionDigits: 0,
  }).format(amount);
  // Output: "Rp 185.000"
}
```
